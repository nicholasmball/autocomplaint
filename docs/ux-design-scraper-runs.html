Scraper Run History — UX Design

Building on the Requirements step (R4, R6). Adds a read-only scraper run history section to the existing admin recipients page. No new pages — extends the current layout with a collapsible panel.

1. Placement — Below recipient table, as a collapsible section

The scraper history lives on the same admin recipients page, below the main table. It's collapsible so it doesn't clutter the primary CRUD workflow. A small status badge in the page header gives at-a-glance info without needing to scroll.

2. Expanded scraper history — Run list

When expanded, shows the most recent 10 runs in a compact list. Each row shows date, trigger type, counts, and a toggle to view details.

5. User flow walkthrough

  1. Admin opens recipients page — sees the header badge "Last scraped X days ago" with colour-coded dot. Scraper history section is collapsed below the table.
  2. Admin clicks "Scraper History" — section expands, showing up to 10 most recent runs. Each row shows trigger type (cron/manual), timestamp, and summary counts.
  3. Admin clicks a run row — detail table expands inline, showing per-company results: name, status (verified/updated/not_found/error), email found, and confidence level.
  4. Admin clicks the row again — detail collapses back to the summary row.
  5. Admin clicks "Scraper History" header again — entire section collapses.
Keyboard: All expand/collapse triggers are <button> elements, focusable and activated with Enter/Space. Detail tables use proper aria-expanded attributes. Colour is never the sole indicator — status text is always present alongside colour coding.

6. Mobile / responsive considerations

  • Header badge: On small screens, the "Last scraped" badge wraps below the title.
  • Run list: Summary counts stack vertically on narrow viewports (processed, updated, errors each on their own line).
  • Detail table: On mobile, switches to a card layout — each company as a stacked card with label: value pairs instead of a horizontal table.
  • Expand/collapse: Touch targets are full-width rows (min 44px height), meeting WCAG touch target guidelines.
