Dashboard badges: Response status appears as a second badge next to the delivery status. Per AC #8, overdue follow-ups get a red "overdue" badge (follow_up_date < today). Complaints in draft/generated status don't show response badges.

3. Interaction Flow

Entry: User views a delivered/sent complaint → tracker section visible at bottom

Status change: Click a status pill → relevant fields animate in (200ms ease-out) → previous fields animate out

Save: Click Save → button shows "Saving..." → PATCH request → "Saved ✓" for 3s

Error: If save fails → red inline error "Failed to save — please try again."

Dashboard: Response badge updates on next page load (no real-time needed)

Overdue: If follow_up_date < today and status is awaiting/no_response/escalated → red "overdue" badge on dashboard

Keyboard nav: Status pills are focusable via Tab, selectable via Enter/Space. Form fields follow standard tab order. Save button is last in tab order.
