Company Autocomplete — UX Design

Building on the Requirements step. Covers: autocomplete dropdown, loading/empty states, keyboard navigation, and mobile layout.

1. Initial State — Company Selected, No Input Yet

Initial state: Same layout as before. Autocomplete only activates after 2+ characters (AC #2). Email field remains separate and editable (AC #6).

2. Typing — Results Found

Results dropdown: Each row shows name (bold), sector (small gray), and email (smallest gray). First item is highlighted (AC #9 — keyboard). Max 10 results (AC #3). Dropdown appears below input, full width. Arrow keys move highlight, Enter selects, Escape closes.

3. Loading State

Loading: Spinner in the right side of the input while the debounced search is in flight (AC #8). No dropdown shown until results arrive.

4. No Results Found

No matches found — enter details manually
Empty state: Non-interactive message (AC #5). Input remains editable — user continues typing their custom company name (AC #6). Dropdown dismisses on blur or further typing below 2 chars.

5. After Selection — Email Auto-filled

Post-selection: Company name fills the input, dropdown closes, email field auto-populates (AC #4). User can still edit both fields manually (AC #6). The email only auto-fills if the company has one — otherwise it stays empty.

6. Regulator Autocomplete — Same Component

  • Ofcom
    Ofcom — Telecoms

Reusable component: Same `RecipientAutocomplete` with `type="regulator"` prop (AC #7). Subtitle shows abbreviation + sector. Searches both name and abbreviation fields.

7. Interaction Flow

Trigger: User types 2+ characters → 300ms debounce → API call (AC #2, #8)

Results: Dropdown opens with up to 10 matches (AC #3)

Selection: Click or Enter → name fills input, email auto-populates, dropdown closes (AC #4)

No match: "No matches found" message, input stays editable (AC #5, #6)

Dismiss: Click outside, Escape, or clearing input below 2 chars closes dropdown

Keyboard: Tab to input → type → Arrow Down/Up to navigate → Enter to select → Escape to close (AC #9)

8. Accessibility

ARIA: Input has role="combobox", aria-expanded, aria-controls, aria-activedescendant, aria-autocomplete="list"

Listbox: Dropdown has role="listbox", each item has role="option" with aria-selected

Active item: Scrolls into view on arrow key navigation

Screen reader: Announces "X results available" when dropdown opens (via aria-live region)
