Complaint reference The ref code (e.g. "AC-2F8A") is derived from the first 4 hex characters of the complaint UUID. This is for the user's own reference only — not used for any system lookup. It gives the user something tangible to remember.

Behaviour

  • localStorage draft is cleared on successful delivery
  • "View in complaint history" navigates to /dashboard (scrolls to the new complaint)
  • "Start new complaint" calls wizard reset() and returns to Step 1
  • Copy/mailto links at the bottom repeat the delivery action without changing status

Design decisions

  • Category icons: Each category maps to a simple unicode icon in a tinted circle. Keeps it lightweight without an icon library.
  • Status badges: Draft (grey), Generated (blue), Reviewed (amber), Delivered (green) — colour-coded for quick scanning.
  • Click target: Entire row is clickable, navigates to /dashboard/complaints/[id].
  • No pagination needed for MVP: Users won't have hundreds of complaints initially. If needed later, add "Load more" button.
  • Server component: Dashboard fetches complaints server-side for fast initial render.

Responsive Behaviour

All new screens must work on mobile (375px+), tablet (768px+), and desktop (1024px+).

Mobile (375px+)
Complaint rows: stack badge below meta text.
Detail page: single column, full-width buttons.
Timeline: horizontal with smaller dots, labels hidden on <400px.
Tablet (768px+)
Complaint rows: badge and chevron inline.
Detail page: same as desktop but narrower padding.
Delivery buttons: side-by-side (existing 2-col grid).
Desktop (1024px+)
Max-width container (existing max-w-4xl).
All elements comfortably spaced.
Full timeline labels visible.

Accessibility (WCAG 2.1 AA)

Checklist for implementation.

  • All status badges use both colour AND text labels (not colour-only)
  • Complaint rows are keyboard-navigable (focusable, Enter/Space to activate)
  • Success overlay announced via aria-live="polite" region
  • Timeline steps have aria-label describing status (e.g. "Draft: completed")
  • "Copy to clipboard" button has aria-live feedback ("Copied to clipboard" announced)
  • Back link on detail page has aria-label="Back to complaint history"
  • Empty state CTA is a real link/button, not just styled text
  • All colour contrast ratios meet 4.5:1 minimum (badges, muted text verified)
  • Form inputs maintain existing focus ring styles (ring-2 ring-blue-600)
  • Tab order follows visual order on all screen sizes
